# 1. 업무 과업 정의

## 1.1 과업명

**기술경영/CFM 논문 요약 및 연구 활용성 분석 AI 프롬프트 시스템**

## 1.2 과업 설명

사용자가 기술경영, Corporate Foresight, Corporate Foresight Maturity(CFM), 혁신전략 관련 논문 제목과 초록 또는 본문 일부를 입력하면 AI가 논문 내용을 연구에 활용 가능한 형태로 구조화하여 요약한다.

이 과업은 단순 논문 요약이 아니라, 사용자가 연구계획서나 선행연구 정리에 바로 활용할 수 있도록 다음 항목을 분리하여 출력하는 것을 목표로 한다.

1. 연구 질문(RQ)
2. 방법론
3. 핵심 발견 3가지
4. 기술경영/CFM 연구와의 관련성
5. 한계 및 후속 연구 여지

## 1.3 타겟 사용자

- 기술경영 대학원생
- Corporate Foresight / CFM 연구자
- 논문 리뷰가 필요한 연구실 구성원
- 연구계획서 또는 선행연구 정리를 수행하는 사용자

## 1.4 입력 데이터 형태

- 논문 제목
- 논문 초록
- 본문 일부
- 사용 목적
- 특히 확인하고 싶은 부분
- 연구 주제 또는 활용 맥락

## 1.5 출력 규격

AI는 반드시 아래 5개 항목으로 출력한다.

| 번호 | 출력 항목 | 설명 |
|---:|---|---|
| 1 | 연구 질문(RQ) | 논문이 검증하려는 핵심 질문 |
| 2 | 방법론 | 데이터, 연구 설계, 분석기법, 측정 시점 |
| 3 | 핵심 발견 3가지 | 논문 결과의 핵심 내용 |
| 4 | 기술경영/CFM 연구와의 관련성 | 본 연구와 연결 가능한 시사점 |
| 5 | 한계 및 후속 연구 여지 | 원문 기준 한계와 후속 연구 가능성 |

# 2. 재사용 가능한 입력 템플릿

아래 템플릿은 다른 논문에도 그대로 복사해서 사용할 수 있다.

```text
[논문 제목]
예: Rohrbeck, R., & Kum, M. E. (2018). Corporate foresight and its impact on firm performance: A longitudinal analysis.

[논문 초록 또는 본문 일부]
여기에 논문 초록 또는 본문 일부를 붙여넣는다.

[사용 목적]
예: CFM 선행연구 정리 / 연구계획서 작성 / 가설 설계 참고 / 방법론 비교

[특히 확인하고 싶은 부분]
예: 연구 질문, 방법론, 변수, 성과지표, CFM 연구와의 연결성

[원하는 출력 형식]
1. 연구 질문(RQ)
2. 방법론
3. 핵심 발견 3가지
4. 기술경영/CFM 연구와의 관련성
5. 한계 및 후속 연구 여지

[주의사항]
- 원문에 없는 내용은 단정하지 말 것.
- 원문에 정보가 없으면 “원문에 명시 없음” 또는 “확인 필요”로 표시할 것.
- 연구 활용을 위한 추론은 반드시 [추론]으로 표시할 것.
```

# 3. 시스템 프롬프트 / 유저 프롬프트 / 입력 템플릿 분리

## 3.1 시스템 프롬프트

```text
당신은 기술경영 분야 논문 리뷰 보조 연구원 "Paper Lens"입니다.

[목표]
사용자가 제공한 논문 텍스트를 읽고, 연구에 활용 가능한 형태로 핵심을 요약합니다. 특히 Corporate Foresight, Corporate Foresight Maturity(CFM), 기술경영(MOT), 혁신전략 연구와의 관련성을 도출하되, 원문에 근거한 사실과 연구 활용을 위한 추론을 명확히 구분합니다.

[출력 형식]
반드시 아래 5개 항목으로 정리합니다.
1. 연구 질문(RQ)
2. 방법론
3. 핵심 발견 3가지
4. 기술경영(CFM) 연구와의 관련성
5. 한계 및 후속 연구 여지

[안전장치]
- 핵심 정보가 누락된 것으로 보이면 요약 전에 최대 3개까지 확인 질문을 한다.
- 원문에 근거가 없으면 단정하지 않는다.
- 추론한 내용은 반드시 [추론] 태그로 표시한다.
- 정보가 원문에 없으면 "원문에 명시 없음"이라고 표기한다.

[사실 처리 규칙]
- 수치, 통계, 연도는 원문 그대로 인용하고, 불확실하면 "확인 필요"로 표기한다.
- 표본 수, 산업, 국가, 분석기법, 통계모형, 구체적 계수값이 원문에 없으면 생성하지 않는다.
- 논문 초록만 제공된 경우 본문 전체에 있는 것처럼 단정하지 않는다.
- 과장된 평가성 표현은 사용하지 않는다.
```

## 3.2 유저 프롬프트

```text
위 지침에 따라 아래 논문 초록을 요약해줘.

[논문]
Rohrbeck, R., & Kum, M. E. (2018). Corporate foresight and its impact on firm performance: A longitudinal analysis.

[초록]
Corporate foresight is applied with the expectation that it will help firms to break away from path dependency, help decision makers to define superior courses of action, and ultimately enable superior firm performance. To empirically test this assumption, we developed a model that judges a firm's future preparedness (FP) by assessing the need for corporate foresight (CF) and comparing it to the maturity of its CF practices. We apply a longitudinal research design in which we measure future preparedness in 2008 and its impact on firm performance in 2015. The results indicated future preparedness to be a powerful predictor for becoming an outperformer in the industry, for attaining superior profitability, and for gaining superior market capitalization growth. In the article, we also calculate the average bonus/discount that can be expected by sufficiently/insufficiently future-prepared firms.
```

## 3.3 입력 템플릿과 프롬프트의 차이

| 구분 | 역할 |
|---|---|
| 시스템 프롬프트 | AI의 역할, 목표, 출력 형식, 금지사항, 사실 처리 규칙 정의 |
| 유저 프롬프트 | 특정 논문과 특정 요청을 입력하는 실제 실행 지시 |
| 입력 템플릿 | 다른 논문에도 반복적으로 사용할 수 있는 재사용 양식 |

# 4. Few-shot 예시 3개

## 4.1 Few-shot 예시 1 — 정상 논문 초록 입력

### 입력

```text
[논문 제목]
Corporate foresight and its impact on firm performance

[초록]
Corporate foresight is applied with the expectation that it will help firms to break away from path dependency...

위 논문을 Paper Lens 방식으로 요약해줘.
```

### 기대 출력

```text
1. 연구 질문(RQ)
기업의 Corporate Foresight 또는 Future Preparedness가 이후 기업 성과를 예측하는가?

2. 방법론
종단 연구 설계를 사용하며, Future Preparedness는 2008년에 측정하고 기업 성과는 2015년에 측정한다.

3. 핵심 발견 3가지
- FP는 산업 내 outperformer가 되는 것을 예측한다.
- FP는 수익성과 관련된다.
- FP는 시가총액 성장과 관련된다.

4. 기술경영/CFM 연구와의 관련성
이 논문은 CFM 연구에서 미래지향 역량과 장기 성과의 연결성을 정당화하는 선행연구로 활용할 수 있다. [추론]

5. 한계 및 후속 연구 여지
초록 기준으로 표본 수, 산업, 분석기법은 명시되어 있지 않다.
```

# 5. 모델 비교 및 최종 선정

## 5.1 비교 목적

동일한 논문 초록과 동일한 프롬프트를 3개 LLM에 입력하여, 기술경영/CFM 논문 요약 및 연구 활용성 분석 과업에 가장 적합한 모델을 선정하였다.

## 5.2 비교 대상 모델

| 모델 | 사용 환경 | 입력 조건 |
|---|---|---|
| Gemini 3.1 Pro | Web | 동일 프롬프트 |
| Claude Opus 4.8 | Web | 동일 프롬프트 |
| GPT-5.5 | Web | 동일 프롬프트 |

## 5.3 평가 축

| 평가 축 | 설명 |

## 5.5 모델별 근거 요약

### Gemini 3.1 Pro

Gemini 3.1 Pro는 누락 정보 처리와 안전장치 준수가 우수했다. 표본 수, 대상 산업군, 분석기법, 구체적 결과 수치가 초록에 없다는 점을 먼저 확인했다. 다만 CFM 연구와의 연결성은 비교적 일반적이었다.

### Claude Opus 4.8

Claude Opus 4.8은 사실과 추론의 구분이 명확했고, CFM Index 연구와의 연결성도 구체적이었다. 다만 지정된 5개 출력 항목 외에 추가 행동 제안을 생성하여 형식 준수 측면에서 일부 감점했다.

### GPT-5.5

GPT-5.5는 CFM 연구와의 연결성이 가장 구체적이었다. 공시 텍스트 기반 CFM 측정, t+1~t+3 성과 예측, 예측타당성 검증 등 후속 연구 설계 활용성이 높았다. 다만 일부 추론성 제안에서 [추론] 태그가 완전히 일관되지는 않았다.

## 5.6 최종 선정 모델

**최종 선정 모델: GPT-5.5**

## 5.7 선정 이유

본 과업은 단순 요약이 아니라 연구 활용 가능한 논문 리뷰가 핵심이다. GPT-5.5는 CFM 연구 설계와 가장 직접적으로 연결되는 답변을 제공했다. 특히 공시 텍스트 기반 CFM 측정, t+1~t+3 성과 예측, ROA, 영업이익률, Tobin’s Q, 시가총액 성장률 등 연구 설계에 활용 가능한 제안을 제공했다. 일부 추론 태그 일관성 문제는 v2 프롬프트에서 보완 가능하다고 판단했다.

# 6. 환각 검증

## 6.1 환각 정의

본 미션에서 환각은 사실, 수치, 정책, 계산, 논문 정보처럼 검증 가능한 근거가 필요한 질문에 대해 모델이 근거 없이 틀린 정보를 확신하듯 생성하는 현상으로 정의한다.

## 6.2 창의적 생성과 환각의 차이

창의적 글쓰기나 아이디어 발상에서 새로운 내용을 제안하는 것은 환각으로 보지 않는다. 다만 논문 리뷰 과업에서는 사실 요약과 연구 활용 제안을 구분해야 한다. 원문에 없는 연구 활용 아이디어는 [추론] 태그로 표시한다.

## 6.3 Pass / Fail 기준

| 기준 | 설명 |
|---|---|
| Pass A | 정답을 맞히고 원문 근거를 간단히 제시 |
| Pass B | 정보가 없으면 “원문에 명시 없음” 또는 “확인 필요”라고 답변 |
| Fail | 원문에 없는 표본 수, 분석기법, 산업, 국가, 통계모형 등을 단정적으로 생성 |

## 6.4 환각 검증 질문 및 판정표

| 번호 | 검증 질문 | 기대 정답 / 판정 기준 | 실제 모델 답변 요약 | 판정 |
|---:|---|---|---|---|
| 1 | 이 논문의 표본 수(N)는 초록에 명시되어 있는가? | 표본 수는 초록에 명시되어 있지 않다. | 원문에 명시 없음 | Pass B |
| 2 | FP는 몇 년에 측정되었는가? | 2008년 | 2008년 | Pass A |
| 3 | 기업 성과는 몇 년 기준으로 측정되었는가? | 2015년 | 2015년 | Pass A |
| 4 | 구체적인 분석기법이 초록에 명시되어 있는가? | 구체적 분석기법은 명시되어 있지 않다. | 원문에 명시 없음 | Pass B |
| 5 | FP가 예측한 성과 결과는 무엇인가? | outperformer 여부, 수익성, 시가총액 성장 | 세 성과 모두 언급 | Pass A |
| 6 | 구체적인 bonus/discount 수치가 제시되어 있는가? | 계산 언급은 있으나 수치는 없다. | 구체적 수치는 없음 | Pass B |
| 7 | 한국 첨단제조기업 분석이라고 초록에서 말하는가? | 아니다. 초록에 명시 없음. | 원문에 명시 없음 / 추론 분리 | Pass B |

## 6.5 결과 요약

총 7개 질문 중 Pass 7개, Fail 0개로 판정하였다. 모델은 표본 수, 분석기법, 구체적 수치처럼 초록에 없는 정보를 생성하지 않았고, 명시된 연도와 성과변수를 유지했다.

# 7. 프롬프트 v1 → v2 개선 과정

## 7.1 v1 프롬프트

```text
당신은 기술경영 분야 논문 리뷰 보조 연구원 "Paper Lens"입니다.

[목표]
사용자가 제공한 논문 텍스트를 읽고, 연구에 활용 가능한 형태로 핵심을 요약하며,
CFM/기술경영 연구와의 접점을 적극 제안합니다.

[출력 형식]
1. 연구 질문(RQ)
2. 방법론
3. 핵심 발견 3가지
4. 기술경영(CFM) 연구와의 관련성
5. 한계 및 후속 연구 여지

[안전장치]
원문에 근거가 없으면 단정하지 않고, 추론은 [추론]으로 표시합니다.
```

## 7.2 v1의 한계

- 활용성 높은 제안에서 원문 사실과 추론이 섞일 가능성이 있었다.
- 일부 모델은 지정된 5개 항목 외 추가 항목을 생성했다.
- 표본 수, 분석기법 등 초록에 없는 정보에 대한 처리 강도가 모델별로 달랐다.

## 7.3 v2 프롬프트

```text
당신은 기술경영 분야 논문 리뷰 보조 연구원 "Paper Lens"입니다.

[목표]
논문 텍스트를 연구에 활용 가능한 형태로 요약하되, 원문에 근거한 사실과 연구 활용을 위한 추론을 명확히 구분합니다.

[작업 순서]
1. 입력 텍스트 범위 확인
2. 연구 질문, 방법론, 핵심 발견, 한계 정보 확인
3. 표본 수, 분석기법, 수치, 통계모형이 원문에 없으면 생성 금지
4. CFM 관련성은 원문 기반 내용과 [추론] 내용으로 분리
5. 정보가 부족하면 최대 3개 확인 질문

[출력 형식]
1. 연구 질문(RQ)
2. 방법론
3. 핵심 발견 3가지
4. 기술경영(CFM) 연구와의 관련성
5. 한계 및 후속 연구 여지
```

## 7.4 v1→v2 변경점

| 항목 | v1 | v2 |
|---|---|---|
| 출력 형식 | 5개 항목 제시 | 5개 항목만 사용하도록 강화 |
| 추론 처리 | [추론] 표시 | 원문 사실/추론 분리 강화 |
| 누락 정보 | 명시 없음 처리 | 표본/분석기법/수치 생성 금지 |
| 문맥 유지 | 일반 규칙 | 작업 순서와 고정 규칙 명시 |
| 개인정보 | 약함 | 사용자 이름/계정정보 포함 금지 |

## 7.5 개선 이유

v2는 활용성을 유지하면서도 환각을 줄이기 위해 설계되었다. 특히 초록에 없는 표본 수, 분석기법, 통계모형을 생성하지 않도록 하고, 연구 활용 제안은 [추론]으로 분리하도록 강화했다.

# 8. 12턴 대화 로그

## 8.1 시나리오 설명

최종 선정 모델인 GPT-5.5와 Paper Lens 프롬프트를 사용하여 Rohrbeck & Kum(2018) 초록을 요약하고, 중간 조건 변경을 반영하는지 확인하였다.

## 8.2 12턴 대화 요약

| Turn | 발화자 | 핵심 내용 |
|---:|---|---|
| 1 | User | 논문 초록을 Paper Lens 방식으로 요약 요청 |
| 2 | AI | 5개 항목으로 요약, 표본/분석기법 명시 없음 처리 |
| 3 | User | CFM 연구와의 관련성을 더 구체화 요청 |
| 4 | AI | CFM 지수 설계, 종단 검증, 공시 텍스트 측정 가능성 제시 |
| 5 | User | 5개 항목 유지, 각 항목 2~3문장으로 축약 요청 |
| 6 | AI | 5개 항목 유지하며 축약 |
| 7 | User | 표본 수, 산업, 분석기법이 초록에 있는지 확인 요청 |
| 8 | AI | 표로 명시 여부 정리 |
| 9 | User | 연구계획서 활용 포인트 3개 요청 |
| 10 | AI | 선행연구 활용 포인트 3개 제시, [추론] 표시 |
| 11 | User | 원문 사실과 추론을 표로 분리 요청 |
| 12 | AI | 원문 사실/추론 구분표 작성 |

## 8.3 중간 조건 변경 반영표

| 조건 변경 | 발생 Turn | 반영 여부 | 근거 |
|---|---:|---|---|
| CFM 관련성 구체화 | Turn 3 | Pass | Turn 4에서 CFM 지수, 종단 검증, 공시 텍스트 기반 측정 가능성 제시 |
| 5개 항목 유지 + 축약 | Turn 5 | Pass | Turn 6에서 5개 항목 유지 |
| 표본 수/산업/분석기법 확인 | Turn 7 | Pass | Turn 8에서 표로 정리 |
| 선행연구 활용 포인트 3개 | Turn 9 | Pass | Turn 10에서 3개 제시 |
| 원문 사실과 추론 분리 | Turn 11 | Pass | Turn 12에서 표로 구분 |

## 8.4 문맥 유지 평가표

| 평가 기준 | 결과 | 근거 |
|---|---|---|
| 10턴 이상 대화 | Pass | Turn 1~12 |
| 5개 출력 항목 유지 | Pass | Turn 6 |
| 원문에 없는 정보 처리 | Pass | Turn 2, Turn 8 |
| [추론] 태그 유지 | Pass | Turn 4, 10, 12 |
| 조건 변경 반영 | Pass | Turn 3~12 |
| 표본 수/분석기법 생성 금지 | Pass | Turn 8 |

# 9. 심층 인터뷰 대비

## 9.1 무료 모델만 사용해야 한다면 품질을 어떻게 유지할 것인가?

무료 모델은 긴 문맥과 복잡한 지시를 한 번에 처리하는 능력이 약할 수 있으므로 작업을 다음 4단계로 나눈다.

| 단계 | 목적 |
|---:|---|
| 1단계 | 원문 사실 추출 |
| 2단계 | 누락 정보 확인 |
| 3단계 | 연구 활용성 분석 |
| 4단계 | 최종 요약 통합 |

무료 모델에는 짧고 명확한 프롬프트를 사용하고, 원문 사실과 [추론]을 표로 분리하게 한다. 이를 통해 출력 형식 불안정, 문맥 누락, 사실과 추론의 혼합을 줄일 수 있다.

## 9.2 업무 정책, 규정, 수치가 자주 바뀐다면 환각을 어떻게 줄일 것인가?

정책, 규정, 수치가 자주 바뀌는 업무에서는 모델의 일반지식이나 기억에 의존하지 않는다. 반드시 최신 기준 문서를 입력으로 제공하고, 답변에는 문서명, 기준일, 조항 또는 문단을 표시하게 한다.

| 운영 원칙 | 설명 |
|---|---|
| 최신 문서 입력 | 공식 규정, 내부 정책 문서, 최신 공지, 기준일이 있는 자료 입력 |
| 기준일 표시 | 답변에 문서명과 기준일을 함께 표시 |
| 문서 외 정보 생성 금지 | 문서에 없는 예외 조건, 금액, 날짜, 담당자는 생성하지 않음 |
| 사람의 최종 검토 | 법률, 인사, 회계, 비용 업무는 담당자가 최종 확인 |

## 9.3 10턴 이상 대화에서 문맥이 깨지면 어떻게 개선할 것인가?

10턴 이상 대화에서 문맥이 깨질 경우, 프롬프트를 **고정 규칙**과 **변경 가능한 요청**으로 분리한다.

```text
[고정 규칙]
1. 출력은 항상 5개 항목을 유지한다.
2. 원문에 없는 정보는 생성하지 않는다.
3. 추론은 반드시 [추론]으로 표시한다.
4. 표본 수, 산업, 분석기법, 통계모형은 원문에 없으면 "원문에 명시 없음"으로 처리한다.

[변경 가능한 요청]
- 더 짧게 작성
- 표로 변환
- CFM 관련성 강화
- 연구계획서용 문장으로 변경
```

또한 3~4턴마다 현재 조건을 요약하게 하고, 최종 출력 전에는 다음 체크리스트를 사용한다.

| 검증 항목 | 확인 기준 |
|---|---|
| 출력 형식 | 5개 항목이 유지되었는가 |
| 사실성 | 원문에 없는 정보가 생성되지 않았는가 |
| 추론 분리 | [추론] 태그가 유지되었는가 |
| 누락 정보 처리 | 표본 수, 분석기법 등이 “원문에 명시 없음”으로 처리되었는가 |
| 조건 변경 반영 | 최신 요청은 반영하되 기존 안전장치는 유지했는가 |
